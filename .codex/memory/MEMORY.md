# Memory index

## How this memory works

This repository uses `memory-playbook-cx`: `MEMORY.md` is a compact rule and domain map,
domains hold current status, and episodes preserve only reasoning that future work cannot
recover cheaply from Git. Before the final response of substantive work, run the capture gate
from the installed skill. `Do not remember` or `temporary` opts the current task out.

Required behavior belongs in `AGENTS.md`. Never copy Codex-managed global memories into this
repository without explicit owner approval.

## Domain map

- [project-map](project-map.md) — stable repository facts, verification, and entry points.
- [playbook](playbook.md) — owner-approved quality and presentation expectations.

## Global gotchas

- A skill alone cannot guarantee end-of-task execution; Bootstrap must add the checked-in
  `AGENTS.md` closeout rule.
- Codex-managed memories are generated state and must not be edited or copied into this public
  repository automatically.
